## 2. 사용되지 않는 코드 분석 및 제거

### 2.1 분석 원칙

**전제 조건:**
- 관리자용 웹앱은 **개발이 완료된 상태**
- 더 이상 추가할 기능이 없음
- 현재 작동하는 기능 이외의 코드는 모두 "쓸데없이 놀고 있는 코드"

**제거 기준:**
- 웹앱에서 실제로 호출되지 않는 함수
- UI에서 사용되지 않는 기능
- 구현되지 않은 기능의 백엔드 함수

### 2.2 Code.gs에서 제거된 함수 상세 분석

#### 2.2.1 `normalizePackageId(packageId)` 함수

**위치:** Code.gs 라인 1711-1720 (제거 전)

**원래 목적:**
- 패키지 ID를 정규화하는 유틸리티 함수
- 예: `PKG1` → `PKG00001`, `PKG123` → `PKG00123`

**왜 사용되지 않는가:**
1. **호출 위치 없음**: 전체 코드베이스에서 이 함수를 호출하는 곳이 전혀 없음
2. **유사 함수와의 비교**: 
   - `normalizeTemplateId()`는 실제로 사용됨 (`createNewSurvey()` 함수 내에서 사용)
   - `normalizePackageId()`는 정의만 되어 있고 어디서도 호출되지 않음

**맥락에서 벗어난 이유:**
- 개발 과정에서 템플릿 ID 정규화는 필요했지만, 패키지 ID 정규화는 실제로 사용되지 않았음
- 패키지 ID는 `generatePackageId()` 함수에서 이미 정규화된 형식(`PKG000001`)으로 생성되므로 추가 정규화가 불필요했음

**제거 판단:** ✅ **제거 적절** - 어디서도 호출되지 않는 순수 데드 코드

**위치:** Code.gs `getActiveSurveysV3(filterByNickname)` 함수 (라인 646)

**왜 사용되지 않는가:**
1. **함수 내부 미사용**: 함수 본문에서 `filterByNickname`을 참조하는 코드가 없음
2. **호출 시점**: 
   - `getActivePackages()` 함수에서 `getActiveSurveysV3()`를 호출할 때 파라미터를 전달하지 않음
   - 실제 호출: `return getActiveSurveysV3();` (파라미터 없음)

**맥락에서 벗어난 이유:**
- 초기 설계에서 닉네임 필터링 기능을 계획했으나, 실제 구현 단계에서 필요하지 않아 구현되지 않음
- 파라미터만 남아있고 실제 로직은 구현되지 않은 상태

**제거 판단:** ✅ **제거 적절** - 사용되지 않는 파라미터, 함수 시그니처 정리 필요

#### 2.2.3 `copyTemplate(templateId)` 함수

**위치:** Code.gs 라인 2024-2062 (제거 전)

**원래 목적:**
- 기존 템플릿을 복사하여 새 템플릿 생성
- 템플릿 데이터를 복사하고 이름에 " (복사본)" 추가하여 새 템플릿 생성

**왜 사용되지 않는가:**
1. **index.html에서 호출 없음**: 
   - 템플릿 복사 기능은 UI에서 "기존 템플릿 복사" 모드로 구현되어 있음
   - 하지만 이 모드는 `getTemplateById()`로 템플릿을 가져와서 폼에 채우고, `createTemplate()`로 새로 생성하는 방식
   - `copyTemplate()` 함수를 직접 호출하지 않음

2. **실제 구현 방식**:
   ```javascript
   // index.html의 실제 구현
   selectCopyTemplate(templateId) {
     google.script.run
       .getTemplateById(templateId)  // 템플릿 가져오기
       // ... 폼에 데이터 채우기
       // ... createTemplate() 호출
   }
   ```

**맥락에서 벗어난 이유:**
- 템플릿 복사 기능은 필요했지만, 구현 방식이 달라짐
- UI에서 직접 복사하는 방식(`getTemplateById()` → 폼 채우기 → `createTemplate()`)을 선택
- `copyTemplate()` 함수는 구현되었지만 실제로는 사용되지 않는 대체 경로

**제거 판단:** ✅ **제거 적절** - 실제 사용되는 경로가 다른 방식으로 구현되어 있음

   - 패키지 수정 기능(`editPackage()`)이 TODO 상태로 구현되지 않음

**맥락에서 벗어난 이유:**
- 패키지 수정 기능을 계획했으나 실제로 구현되지 않음
- 패키지 수정 UI가 없으므로 패키지 상세 조회 함수도 필요 없음

**제거 판단:** ✅ **제거 적절** - 패키지 수정 기능이 구현되지 않아 사용되지 않음

2. **수동 호출 없음**:
   - UI에서 이 함수를 호출하는 버튼이나 기능이 없음
   - 관리자가 수동으로 실행할 방법이 없음

**맥락에서 벗어난 이유:**
- 자동 완료 기능을 계획하고 구현했으나, 실제로 트리거를 설정하지 않음
- 또는 수동 실행 기능을 추가하지 않음
- 개발이 완료된 상태이므로 더 이상 설정할 계획이 없음

**제거 판단:** ✅ **제거 적절** - 트리거가 설정되지 않아 실행되지 않음

### 1.1 시스템 용어 정리

#### 핵심 개념

**템플릿 (Template)**
- 정의: 개별 설문지/심리검사 하나의 단위
- 예시: BDI 우울척도, BAI 불안척도, DTI-30-R 등
- 저장 위치: `TEMPLATE_SHEET_ID` 스프레드시트
- 시트 구성: '일반 척도', '자기측면검사', '정서자극판단과제'

**패키지 (Package)**
- 정의: 연구 목적에 따라 여러 템플릿을 묶은 설문 세트
- 예시: PKG000001 = [자기측면검사 + BDI + BAI + 스트레스 척도]
- 저장 위치: `SURVEY_LIST_SHEET_ID` 스프레드시트
- 속성:
  - 패키지 코드 (PKG000XXX)
  - 템플릿 ID 목록 (쉼표 구분)
  - 패키지 타입: 'L'(종단연구) / 'X'(횡단연구)
  - 시작/종료일
  - 응답 시트 URL

**응답 흐름**
```
피검자 로그인
→ 활성 패키지 목록 확인
→ 패키지 코드 검증
→ 템플릿 목록 표시
→ 개별 템플릿 응답 화면 (구현 예정)
→ 응답 데이터 저장 (구현 예정)
```

## 5. 레거시 데이터 오류 처리 방침

### 5.1 발견된 오류 유형

**1. 빈 option label (BDI 2번 질문)**
```json
{
  "options": [
    {"value": "0", "label": "나는 슬프지 않다"},
    {"value": "1", "label": "나는 슬프다."},
    {"value": "2", "label": "나는 항상 슬프고 기운을 낼 수 없다."},
    {"value": "3", "label": "나는 너무나 슬프고 불행해서 도저히 견딜 수 없다."},
    {"value": "4", "label": ""}    ← 빈 문자열
  ]
}
```

**2. 빈 question text (BDI, SWB, PSS)**
- text 필드가 빈 문자열

### 5.2 처리 방침

**원칙: 피검자용 웹앱에서는 오류 데이터를 있는 그대로 표시**

❌ **하지 않을 것:**
- 빈 label 필터링
- 빈 text 자동 생성
- 유효성 검사로 인한 렌더링 차단

✅ **할 것:**
- 빈 label → 빈 칸으로 라디오 버튼 표시
- 빈 text → 질문 제목 없이 선택지만 표시
- 관리자가 볼 수 있도록 오류 데이터 그대로 노출

**이유:**
- 관리자용 웹앱은 이미 검증 로직 완비 (새 템플릿은 오류 없음)
- 레거시 데이터는 과거 시스템에서 생성된 것
- 피검자가 응답한 역사적 데이터와 일관성 유지
- 오류가 보이면 관리자가 수정하도록 유도

### 3.4 레거시 데이터 처리

**구현된 처리 방침:**
- ✅ 빈 question text → 질문 제목 없이 선택지만 표시
- ✅ 빈 option label → "(내용 없음)" 표시로 오류 노출
- ✅ 필터링 없이 있는 그대로 렌더링

**처리 예시:**
```javascript
// 범주형 질문 렌더링
if (question.text && question.text.trim()) {
  html += `<div>${escapeHtml(question.text)}</div>`;  // 있으면 표시
}
// 없으면 생략 (빈 칸으로 렌더링)

// 빈 라벨 처리
const displayLabel = opt.label || '(내용 없음)';  // 오류 노출
```

## 4. 핵심 기술 사항

### 4.1 템플릿 ID 정규화

```javascript
const normalizedId = String(templateId || '').trim().replace(/-/g, '');
if (templateId === '-1' || normalizedId === 'TPL0001') {
  return getMysideTemplate();
}
if (templateId === '-2' || normalizedId === 'TPL0002') {
  return getEmotionTemplate();
}
```

**처리 방식:**
- `TPL-0001` → `TPL0001` (하이픈 제거)
- `TPL0001` → `TPL0001` (그대로)
- `-1` → 자기측면검사
- `-2` → 정서자극판단과제

### 4.2 패키지 상태 코드

| 코드 | 의미 | 피검자 화면 표시 |
|------|------|-----------------|
| P | Protected (진행중) | ✅ 표시 |
| C | Completed (완료) | ❌ 숨김 |
| H | Hidden (숨김) | ❌ 숨김 |

### 4.3 종단 vs 횡단 연구

**횡단연구 (X):**
- 각 템플릿당 1회만 응답 가능
- 완료한 템플릿은 "응답 완료" 표시 및 버튼 비활성화

**종단연구 (L):**
- 각 템플릿당 여러 번 응답 가능
- 응답 횟수와 마지막 응답 시간 표시
- "응답하기" 버튼 항상 활성화

### 4.4 복수선택 응답 저장

```javascript
// 체크박스 복수선택: 쉼표로 구분하여 저장
const checked = Array.from(container.querySelectorAll('input[type="checkbox"]:checked'))
  .map(cb => cb.value);
AppState.responses[question.id] = checked.join(',');  // 예: "1,3,5"
```
