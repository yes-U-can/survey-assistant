## 2. 범주형 템플릿 지원 완성

### 2.1 범주형 템플릿 UI/UX 개선

**배경:**
- 기존 DevNotes_20251231.md에서 xselect=2 (범주형) 지원 필요성 확인
- DB에 이미 25개의 xselect=2 템플릿이 options 배열과 함께 저장되어 있음
- 기존 구현: 수동으로 value+label 입력 → UI가 복잡하고 보이지 않는 문제

**DB 분석 결과:**
- 모든 범주형 템플릿은 순차적인 value 사용 (0,1,2,3... 또는 1,2,3,4...)
- BDI: 0부터 시작 (0,1,2,3,4)
- 대부분: 1부터 시작 (1,2,3,4,5...)
- 음수 시작값도 허용 필요 (-1, -9 등)

**새로운 UX 설계:**
1. 사용자가 **선택지 개수**와 **시작 값**(정수)을 입력
2. 시스템이 자동으로 순차적인 value 생성 (시작값, 시작값+1, 시작값+2, ...)
3. 사용자는 각 value에 대한 **라벨만** 입력

**예시:**
- 선택지 개수 = 3, 시작 값 = -1 → values: -1, 0, 1
- 선택지 개수 = 5, 시작 값 = 1 → values: 1, 2, 3, 4, 5

**generateOptionLabelsHTML() 함수 (index.html 라인 1377-1391):**

선택지 개수와 시작값을 받아 라벨 입력 필드 자동 생성:

```javascript
function generateOptionLabelsHTML(count, startValue, existingOptions = []) {
  let html = '';
  for (let i = 0; i < count; i++) {
    const value = startValue + i;  // 순차적 value 계산
    const label = existingOptions[i] ? existingOptions[i].label : '';

    html += `
      <div style="display: flex; gap: 0.5rem; margin-bottom: 0.5rem; align-items: center;">
        <span style="font-weight: 600; color: var(--text-secondary); width: 50px; flex-shrink: 0;">
          ${value}:
        </span>
        <input type="text" class="form-input" placeholder="라벨 입력"
               value="${escapeHtml(label)}"
               data-option-label
               data-option-value="${value}"
               style="flex: 1;">
      </div>
    `;
  }
  return html;
}
```

**updateOptionLabels() 함수 (index.html 라인 1393-1418):**

개수 또는 시작값 변경 시 기존 라벨 데이터를 보존하면서 UI 재생성:

```javascript
function updateOptionLabels(questionIndex) {
  const questionItem = document.querySelector(`.question-item[data-index="${questionIndex}"]`);
  const optionsContainer = questionItem.querySelector('.options-container');

  const count = parseInt(questionItem.querySelector('[data-option-count]').value) || 1;
  const startValue = parseInt(questionItem.querySelector('[data-start-value]').value) || 0;

  // 기존 라벨 데이터 보존
  const existingLabels = [];
  questionItem.querySelectorAll('[data-option-label]').forEach(input => {
    existingLabels.push(input.value.trim());
  });

  const existingOptions = existingLabels.map((label, idx) => ({
    value: startValue + idx,
    label: label
  }));

  // UI 재생성
  optionsContainer.innerHTML = generateOptionLabelsHTML(count, startValue, existingOptions);
}
```

## 4. 기술 노트

### 4.1 음수 시작값 지원

**요구사항:**
- 시작값에 -1, -9 같은 음수도 입력 가능
- 음수부터 순차 증가 (예: -1, 0, 1 또는 -9, -8, -7, -6)

**구현:**
- `<input type="number">` 에 min 제약 없음
- `parseInt()` 는 음수 처리 가능
- value 계산: `startValue + i` (i는 0, 1, 2, ...)

**예시:**
```
선택지 개수 = 4, 시작 값 = -1
→ values: -1, 0, 1, 2

선택지 개수 = 3, 시작 값 = -9
→ values: -9, -8, -7
```

### 4.2 기존 데이터 보존

**updateOptionLabels() 로직:**
1. 사용자가 개수 또는 시작값 변경
2. 기존 라벨 입력값을 배열로 수집
3. 새로운 개수/시작값으로 UI 재생성
4. 기존 라벨을 새 위치에 복원 (인덱스 유지)

**효과:**
- 사용자가 개수를 3→5로 늘리면 기존 3개 라벨 유지 + 2개 빈 칸 추가
- 시작값을 1→0으로 변경하면 라벨은 유지되고 value만 변경 (1,2,3 → 0,1,2)

### 4.3 CSS 변수 시스템

**장점:**
- 색상을 한 곳에서 관리 (`:root`)
- 일관된 디자인 유지
- 향후 테마 변경 용이 (다크모드 등)
