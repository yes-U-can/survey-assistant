```javascript
templates.forEach((template, idx) => {
  const templateData = getTemplateById(template.id);
  if (templateData) {
    const templateName = template.name || template.id;

    // 템플릿 설명 구성
    let templateDesc = templateData.description || '';

    // 척도 정보 추가
    const scaleInfoParts = [];
    const minScale = parseInt(templateData.min || '1');
    const maxScale = parseInt(templateData.max || '5');

    if (!isNaN(minScale) && !isNaN(maxScale)) {
      scaleInfoParts.push(`최소값: ${minScale} ~ 최대값: ${maxScale} (단위 간격: 1)`);

      // 척도 라벨 추가
      if (templateData.scaleLabels && typeof templateData.scaleLabels === 'object') {
        const labelParts = [];
        for (let val = minScale; val <= maxScale; val++) {
          const valStr = String(val);
          if (templateData.scaleLabels[valStr]) {
            labelParts.push(`[${val}] 값 설명: ${templateData.scaleLabels[valStr]}`);
          }
        }
        if (labelParts.length > 0) {
          scaleInfoParts.push(labelParts.join('\n'));
        }
      }

      scaleInfoParts.push('묶음형 단순 설문');
    }

    if (scaleInfoParts.length > 0) {
      templateDesc += '\n\n' + scaleInfoParts.join('\n');
    }

    infoData.push([`${idx + 1}. ${templateName}`, templateDesc || '']);
  }
});
```

## 3. 발견된 구조적 문제

### 3.1 템플릿 메타데이터 필드 누락

**배경:**
레거시 데이터 분석 중 현재 DB 구조에서 누락된 필드 발견

#### 레거시 템플릿 메타데이터 (request_sigle_list.csv):

```
[0] no (템플릿 번호)
[1] subject (템플릿 이름)
[2] r_min (최소값)
[3] r_max (최대값)
[4] content (설명)
[5] RType ← 누락
[6] xselect ← 누락
[7] bundle ← 누락
```

#### 현재 템플릿 스프레드시트 구조:

```
[0] templateId ✅
[1] name ✅
[2] description ✅
[3] min ✅
[4] max ✅
[5] scaleLabels (JSON) ✅
[6] questions (JSON) ✅ (options 필드 여부 미확인)
[7] ?
[8] status
```

#### 누락된 필드 상세:

**1. RType (Response Type)**

**의미:**
- B = Basic 리커트 척도 템플릿
- 기타 값: 데이터 오류 (설명 문구가 잘못 입력됨)

**분포:**
- RType='B': 79개 (전체의 대다수)
- 기타: 51개 (데이터 오류)
- RType='1': 1개

**영향:**
- '묶음형 단순 설문' 표시 여부 결정
- 현재 웹앱: 모든 템플릿을 '묶음형 단순 설문'으로 표시 → 큰 문제 없음

**2. xselect (선택지 타입)**

**의미:**
- **xselect=1 (155개)**: 모든 질문이 **같은 척도 공유** (예: 1~5점)
  - 일반적인 리커트 척도
  - 질문: "나는 행복하다", "나는 우울하다"
  - 척도: 모두 "1=전혀 아니다 ~ 5=매우 그렇다"

- **xselect=2 (15개)**: 각 질문마다 **개별 선택지**
  - 카테고리형 질문
  - 예시:
    - "[HYCU] 지원시 후보 대학": 1=서울대, 2=연세대, 3=고려대...
    - "결혼상태": 1=미혼, 2=기혼, 3=이혼, 4=사별, 5=기타
    - "알코올 의존 척도": 각 질문마다 다른 선택지

**Data Migration 처리 로직:**
```javascript
const xselect = template.metadata?.xselect || 1;

    value: opt.value,

```

- **심각**: xselect=2 템플릿은 질문마다 고유한 선택지(options)가 필요
- 현재 웹앱: xselect 정보 없음
- 현재 questions JSON에 options 필드 여부 **미확인**

## 4. 기술 노트

### 4.1 xselect vs 묶음형 단순 설문

**xselect (선택지 구조):**
- xselect=1: 공통 척도 (리커트 척도)
- xselect=2: 개별 선택지 (카테고리형)

**묶음형 단순 설문 (RType/bundle):**
- RType='B' OR bundle='1': 묶음형 설문
- 여러 질문을 하나의 템플릿으로 묶어서 관리
- **xselect와는 독립적인 개념**

**예시:**
- "STAXI-K-10": RType='B', xselect='1', bundle='0' → 묶음형 단순 설문 + 공통 척도
- "[HYCU] 지원시 후보 대학": RType='B', xselect='2', bundle='0' → 묶음형 단순 설문 + 개별 선택지

### 4.2 시트 이름 정확성

**'일반 리커트 척도' 시트 문제점:**
- xselect=2 템플릿(카테고리형)도 포함
- 엄밀히 말해 리커트 척도가 아닌 템플릿도 포함되어 있음

**대안:**
- '일반 설문 템플릿' (가장 포괄적)
- '단순 설문 템플릿' ('묶음형 단순 설문'에서 따옴)
- '설문 템플릿' (가장 간단)

**변경 시 작업:**
- Code.gs 전체에서 `'일반 리커트 척도'` 찾아 바꾸기 (약 15곳)
- 실제 스프레드시트 시트 이름 변경
