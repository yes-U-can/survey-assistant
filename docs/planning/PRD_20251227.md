## 1. 프로젝트 개요

### 1.1 배경
- **기존 시스템**: yesucan.co.kr 웹사이트에서 운영하던 심리 설문조사 시스템
- **웹사이트 폐쇄**: yesucan 도메인 종료로 인해 설문조사 시스템만 분리 필요
- **마이그레이션**: PHP/MySQL 기반 시스템을 Google Apps Script 기반으로 재구현

### 1.2 목적
프로그래밍 문외한인 심리학 교수님들도 쉽게 사용할 수 있는 독립형 설문조사 시스템 개발

### 1.3 핵심 가치
- **간편성**: 비개발자도 설문지 생성/관리 가능
- **독립성**: Google Drive 기반으로 별도 서버 불필요
- **유연성**: 다양한 심리검사 템플릿 지원
- **확장성**: 신규 템플릿 추가 용이

## 4. 핵심 개념

### 4.1 템플릿 (Template)
설문지 양식 정의

**템플릿 유형**:
1. **일반 리커트 척도** (179개)
   - 예: DTI-30-R, FFMQ-39
   - 구조: N점 척도 + 질문 리스트

2. **자기측면검사** (TPL000001)
   - 3단계 특수 구조
   - 상세 요구사항: 섹션 7.2 참조

3. **정서자극판단과제** (TPL000002)
   - 구조 파악 필요

### 4.2 패키지 (Package)
하나 또는 여러 템플릿을 묶어 실시하는 설문

**패키지 유형**:
- **L (Longitudinal)**: 종단연구 - 여러 번 응답 가능
- **X (Cross-sectional)**: 횡단연구 - 1회성 설문

**패키지 상태**:
- **D (Draft/Active)**: 활성화
- **C (Completed)**: 완료
- **P (Protected)**: 보호됨 (레거시 데이터, 읽기 전용)
- **H (Hidden)**: 숨김 (삭제됨)

### 4.3 ID 체계
- **레거시**: 3자리 (T006, PKG010)
- **신규**: 6자리 (TPL000006, PKG000010)
- **변환 이유**: 확장성 (999개 → 999,999개)

### 7.2 자기측면검사 (Self-Aspect Test)

#### 7.2.1 개요
- **목적**: 개인의 다양한 자기 측면 탐색
- **측면 정의**: 역할, 상황, 영역, 특징에 따른 자기 모습
- **예시**: "학생으로서의 나", "일할 때의 나", "착한 나"

#### 7.2.2 관리자 설정
- **단어 개수**: 40개 (기본값, 수정 가능)
- **측면 개수**: 최대 12개 (기본값, 수정 가능)
- **40개 단어 목록**:
  1. 성질하다
  2. 비도덕적이다
  3. 약하다(강하지 못하다)
  4. 즐겁다
  5. 원만하고 둥글둥글하다
  6. 차갑다
  7. 매력이 없다
  8. 소극적이다
  9. 자랑스럽다
  10. 드러내고 싶다
  11. 자신없다
  12. 잘났다
  13. 편하게 이완되어 있다
  14. 도덕적이다
  15. 어둡고 비관적이다
  16. 불성실하다
  17. 따뜻하다
  18. 자신있다
  19. 부끄럽다
  20. 선하다(착하다)
  21. 사랑스럽다
  22. 강하다
  23. 우울하다
  24. 의욕과 에너지가 부족하다
  25. 악하다(못됐다)
  26. 신중하고 조심스럽다
  27. 적극적이다
  28. 유능하다
  29. 뾰족하고 모가 났다
  30. 못났다
  31. 불안하게 긴장되어 있다
  32. 매력적이다
  33. 밝고 낙천적이다
  34. 무책임하다
  35. 사랑스럽지 않다
  36. 충동적이고 부주의하다
  37. 의욕과 에너지가 넘친다
  38. 감추고 싶다
  39. 무능하다
  40. 책임감 있다

#### 7.2.3 피검자 프로세스

##### 시작 화면
**FR-SAT-1**: 소개 화면
- 제목: "나를 구성하는 다양한 측면을 알아보기"
- 내용:
  - 자기측면 정의
  - 분류 기준 예시 (역할, 상황, 영역, 특징)
  - 측면 개수: 2~12개 (강제 아님)
  - 3단계 구조 안내

**FR-SAT-2**: FAQ 화면
- Q1: 자기측면이란? → A: 나를 이루는 여러 모습
- Q2: 나의 모습을 나누어보란 말인가? → A: 최소 2개 이상
- Q3: 측면 수는 몇개까지? → A: 최대 12개
- Q4: 측면 이름을 밝혀야 하나? → A: 2단계에서 직접 입력
- Q5: 수정 가능한가? → A: 전/후 화살표로 수정 가능
- Q6: 중간 저장되나? → A: 로그아웃해도 저장됨

##### 1단계: 단어로 측면 묘사하기

**FR-SAT-3**: 첫 번째 측면 (`step=1, side=1`)
- 제목: "40개의 단어를 이용해 나의 측면들을 묘사하기"
- 설명: "나의 첫번째 측면에 해당되는 단어를 **모두** 고르면 됩니다"
- UI:
  - 40개 단어를 4열 그리드로 배치
  - 각 단어 옆에 체크박스
  - 선택 개수 제한 없음
- 버튼: "다음 →"

**FR-SAT-4**: 두 번째 측면 이후 (`step=1, side=2~12`)
- 설명: "나의 [N]번째 측면을 떠올려 보십시오"
- 중요 안내: "다른 측면에서 사용했던 단어를 다시 사용해도 무방"
- 중단 옵션: "멈춰도 좋습니다. [여기]를 누르면 2단계로 갑니다"
- 버튼: "← 이전", "다음 →"

**데이터 저장**:
```
myside_save 테이블
step = 1
sub_step = 1~12 (측면 번호)
value = "2;5;8;15;22" (선택한 단어 번호를 세미콜론으로 연결)
```

##### 2단계: 측면 이름 붙이기 + 관계 평가

**FR-SAT-5**: Part 1 - 측면 이름 입력
- 1단계에서 선택한 단어들을 측면별로 정리하여 표시
- 각 측면마다 텍스트 입력 박스 제공
- 예시:
  ```
  첫번째 측면
  유능하다
  [___________________] ← 여기에 이름 입력: "일할 때의 나"
  ```

**FR-SAT-6**: Part 2 - 4가지 관계 차원 평가
각 측면 간의 관계를 -3 ~ +3 (7점 척도)로 평가

**1. 비슷한 정도**
```
-3       -2       -1       0       +1       +2       +3
서로 매우              중간              서로 매우
다르다                이다              비슷하다
```

**2. 밀접한 관련성**
```
-3       -2       -1       0       +1       +2       +3
전혀                   중간              매우 밀접히
별개이다              이다              연결되어 있다
```

**3. 조화로운 정도**

### 7.3 정서자극판단과제 (Emotional Stimulus Judgment Task)

**상태**: 구조 파악 완료 ✅ (레거시 시스템 + 논문 기반 분석)

#### 7.3.1 개요
- **목적**: 양극성 장애의 이분법적 정서 판단 양식 측정
- **출처**: 황성훈(2013) 연구 논문에서 개발
- **특징**: 시나리오 기반 정서 평가 (3단계 평정)
- **레거시 시스템**: 7개 시나리오 구현 완료

#### 7.3.2 시나리오 유형 (레거시 시스템 7개)

**시나리오 1 (혼합형)**: 김성환(남, 29세)
- 공무원 시험 준비 → 불합격 통보 → 대기자 명단 통보
- 핵심: 실패 후 희망적 반전

**시나리오 2 (긍정형)**: 한철민(남, 33세)
- 승진 성공 + 20억 입찰 성공 + 아내 임신 7개월 + 건강검진 양호 + 제주도 여행 계획
- 핵심: 다중 긍정 사건

**시나리오 3 (부정형)**: 강소현(여, 26세)
- 불우한 가정(아버지 폭력) → 탈출 → 결혼 → 남편 폭력 재발
- 핵심: 과거 트라우마 재연

**시나리오 4 (혼합형)**: 최수민(여, 34세)
- 패션 디자인 오디션 최종 10개 진출 → 심사위원 평가 2:2 엇갈림
- 핵심: 긍정/부정 평가 공존

**시나리오 5 (부정형)**: 박관수(남, 42세)
- 사업 실패 + 자금난 + 악성종양 가능성 + 아들 오토바이 절도
- 핵심: 다중 부정 사건

**시나리오 6 (긍정형)**: 나영민(여, 50세)
- 아들 의대 합격 + 본인 사이버대학 장학생 합격
- 핵심: 가족과 자신의 성취

**시나리오 7 (부정형)**: 이민정(여, 26세)
- 대학원 재수 준비 → 면접 호평 → 최종 불합격
- 핵심: 희망 후 좌절

**분류 통계**:
- 긍정형: 2개 (시나리오 2, 6)
- 부정형: 3개 (시나리오 3, 5, 7)
- 혼합형: 2개 (시나리오 1, 4)

#### 7.3.3 피검자 프로세스 (3단계 평정)

##### 1단계: 척도 선택

**FR-EST-1**: 판단 척도 유형 선택
- 사용자는 기쁨과 슬픔을 평가하기 위해 두 가지 척도 중 하나를 선택

**옵션 A: 단일 양극 척도**
```
+-----------------------------+
매우 슬픔         중간         매우 기쁨
```

**옵션 B: 두 개의 독립 차원 척도**
```
슬픔 척도:
+-----------------------------------+
전혀 슬프지 않음              매우 슬픔

기쁨 척도:
+-----------------------------------+
전혀 기쁘지 않음              매우 기쁨
```

**측정 지표**:
- 단일 양극 척도 선택 회수 (높을수록 이분법적 사고)

##### 2단계: 감정 거리 평정

**FR-EST-2**: 기쁨/슬픔 선택 및 거리 설정

**Part 1**: 감정 선택
- 사용자는 시나리오를 읽고 "기쁨" 또는 "슬픔" 중 하나를 선택

**Part 2**: 상대적 거리 평정
- 선택한 감정과 반대 감정 사이의 거리를 슬라이더로 설정
- 한쪽 극단(반대 감정)은 고정, 다른 쪽(선택한 감정)은 가변

**UI 예시**:
```
사용자가 "슬픔"을 선택한 경우:

기쁨(고정점) ←-----------[슬라이더]→ 슬픔(가변점)

사용자가 슬라이더를 움직여 "슬픔"의 위치를 결정
→ 기쁨으로부터 얼마나 떨어져 있는지 설정
```

**측정 지표**:
- 척도의 전체 길이를 100으로 환산
- 반대 감정으로부터의 거리 (백분위)
- 거리가 짧을수록 → 양가감정 경향
- 거리가 멀수록 → 극단적 분리

##### 3단계: 자기 생성 분법 척도

**FR-EST-3**: 눈금 생성 및 평정

**UI**:
```
[버튼: 눈금 추가 (최대 5개까지)]
[버튼: 눈금 삭제]
[버튼: 눈금 재설정]

+-----------------------------------+
매우 슬픔      |    |    |    |      매우 기쁨
               ↑ 사용자가 추가한 눈금
```

**설명**:
- 양 극단에 "매우 슬픔", "매우 기쁨" 2개 눈금이 기본 제공
- 사용자는 중간에 최대 5개의 눈금을 추가 가능
- 최종 눈금 개수: 2개(추가 안함) ~ 7개(5개 추가)
- 눈금을 추가한 후, 그 중 하나를 선택하여 자신의 감정 표시

**측정 지표**:
- 총 눈금 개수
- 눈금이 적을수록 → 단분법적 판단 (이분법적 사고)
- 눈금이 많을수록 → 연속적/다분법적 판단

#### 7.3.4 관리자 기능 (레거시 시스템)

- 기등록된 설문 내용 확인

#### 7.3.6 구현 시 고려사항

**UI/UX**:
- 슬라이더는 마우스 드래그로 직관적 조작 가능
- 모바일에서도 터치로 조작 가능하도록 반응형 설계
- 눈금 추가/삭제 버튼은 명확하게 표시
- 시나리오 진행 상황 표시 (1/7, 2/7, ...)

**데이터 검증**:
- 1단계: A 또는 B 중 하나만 선택 (필수)

**측정 지표 계산**:
- 7개 시나리오 × 3단계 = 21개 응답값
- 집계:
  - 단일 양극 척도 선택 빈도 (0~7회)
  - 평균 거리 (0~100)
  - 평균 눈금 개수 (2~7개)
- 유형별 분석:
  - 긍정형 시나리오 (2개) 평균
  - 부정형 시나리오 (3개) 평균
  - 혼합형 시나리오 (2개) 평균
