# Ops Runbook: Special Template Workflow (2026-03-04)

## Scope
- 대상: 플랫폼 어드민 운영자
- 범위: 특수 템플릿 의뢰 접수부터 스토어 거래 반영까지

## 1) Intake
1. 관리자 의뢰 등록 확인 (`/admin` -> 특수 템플릿 의뢰)
2. 동의 체크 확인:
   - MIT 기반 공개 가능 문구 동의
   - 공개/보상 분리 정책 인지
3. 의뢰 상태 `REQUESTED` 확인

## 2) Triage
1. 플랫폼 어드민 콘솔에서 의뢰 검토
2. 상태 전환:
   - `REQUESTED` -> `REVIEWING`
   - 개발 수락 시 `IN_PROGRESS`
3. 범위/일정/크레딧 조건을 `adminNote`에 명시

## 3) Delivery
1. 구현 완료 후 코드 main 반영(공개 저장소 기준)
2. 의뢰 상태 `DELIVERED` 처리
3. 관리자 계정에 템플릿 소유권 반영 확인

## 4) Store Publication
1. 관리자(의뢰자)가 특수 템플릿 스토어 등록
2. 가격/활성 상태 확인
3. 타 관리자 구매 시:
   - 구매자 `SPEND`
   - 판매자 `REWARD` (80%)
   - 플랫폼 수수료 (20%)

## 5) Monitoring
1. 플랫폼 콘솔 `운영 알림` 섹션 확인
2. 임계치 초과 항목 우선 대응:
   - 의뢰 적체
   - 장기 미처리 의뢰
   - 마이그레이션 실패 급증
   - 관리자 총 크레딧 부족

## 6) Incident Handling
1. 결제/정산 이상 징후 시 거래 ID(`referenceId`) 기준 역추적
2. 크레딧 수동 조정은 플랫폼 어드민만 수행
3. 조정 시 메모 필수(원인/승인자/관련 의뢰 ID)
