# UX Journey Reference (2026-03-04)

## 목적
- 최초 진입 화면에서 피검자/연구관리자/플랫폼운영자 여정을 분리해 혼선을 줄인다.
- 기능 나열이 아니라 "역할별 목적 달성" 중심으로 동선을 설계한다.

## 벤치마크한 패턴(개념 레벨)
1. 응답자 빠른 진입 패턴
- 설문 플랫폼 계열(예: Typeform/Google Forms)의 "링크/코드로 즉시 참여" 흐름
2. 관리자 작업대 패턴
- SaaS 운영툴 계열의 "템플릿 -> 운영 -> 결과" 3단계 구조
3. 플랫폼 운영 콘솔 패턴
- 마켓/정산형 SaaS의 "요청 큐 -> 정산 -> 운영 지표" 흐름

## 이번 반영 항목
1. 홈 진입에서 역할 선택을 최우선으로 배치
2. 피검자/관리자 여정을 각각 독립 카드와 3단계 흐름으로 제시
3. 플랫폼 어드민은 별도 운영 여정 카드로 분리
4. 각 여정마다 즉시 진입 CTA를 제공

## 다음 반영 예정
1. 관리자 콘솔 IA 재배치(템플릿/패키지/결과/의뢰/스토어 탭 그룹화)
2. 피검자 모바일 화면에서 "오늘 해야 할 일" 카드 강조
3. 플랫폼 콘솔에서 운영 경고/요청 큐 우선순위 시각화 강화
