## 1. 데이터 마이그레이션

### 1.1 SQL to CSV 추출
- 스크립트: `extract_sql_to_csv.py`
- 입력: yesucan.sql (103MB, EUC-KR)
- 출력: 11개 CSV 파일 (UTF-8)

| 파일명 | 행 수 | 설명 |
|--------|------|------|
| myside_save.csv | 230,506 | 자기측면검사 응답 |
| request_package_list.csv | 42 | 패키지 메타데이터 |
| request_package_spec.csv | 340 | 패키지-템플릿 매핑 |
| request_sigle_list.csv | 182 | 템플릿 메타데이터 |
| request_sigle_spec.csv | 3,352 | 템플릿 질문 목록 |
| request_single_range.csv | 1,236 | 척도 라벨 |
| request_xselect_item.csv | 440 | 개별 선택지 |
| request_member_info.csv | 3 | 회원 정보 항목 정의 |
| request_member_spec.csv | 31,305 | 회원 개별 정보 |
| request_school.csv | 1 | 참여 학교 목록 |
| request_dic_list.csv | 82 | 정서자극판단과제 시나리오 |

### 1.2 템플릿 데이터 처리
1. **JSON 변환**
   - 스크립트: `convert_template_csv_to_json.py`
   - 출력: 179개 JSON 파일 (3개 템플릿 제외 - 질문 없음)

2. **Google Sheets 로드**
   - 스크립트: `LegacyDataLoader.gs` → `loadLegacyTemplates()`
   - 결과: 179개 템플릿 로드

**컬럼 구조:**
- A: 템플릿ID (TPL000XXX)
- B: 템플릿명
- C: 설명
- D: 최소값
- E: 최대값
- F: 척도라벨(JSON)
- G: 질문리스트(JSON)
- H: 생성일
- I: 상태 (P)

**템플릿 타입:**
- xselect=1 (155개): 공통 척도
- xselect=2 (24개): 개별 선택지

### 1.3 패키지 데이터 처리
1. **응답 데이터 분리**
   - 스크립트: `split_myside_by_package.py`
   - 입력: myside_save.csv (230,505행)
   - 출력: 41개 CSV 파일

2. **CSV to Sheets 변환**
   - 스크립트: `CSVtoSheetsConverter.gs`
   - 처리: 3개씩 배치 (GAS 6분 제한 대응)
   - 결과: 41개 스프레드시트 (템플릿별 시트 분리)

3. **패키지 목록 업데이트**
   - 스크립트: `LegacyDataLoader.gs` → `loadLegacyPackages()`
   - 결과: 42개 패키지 로드

**컬럼 구조:**
- 0: 패키지번호 (PKG000XXX)
- 1: 패키지명
- 2: 패키지 설명 (HTML 태그 제거)
- 3: 선택된 템플릿 (쉼표 구분)
- 4: 공유용URL (스프레드시트 URL)
- 5: 생성일
- 6: 시작일
- 7: 종료일
- 8: 타입 (L=종단연구, X=횡단연구)
- 9: 상태 (P)

**처리 로직:**
- request_package_list.csv: 패키지 기본 정보 (이름, 설명, 날짜, diary)
- request_package_spec.csv: 패키지별 템플릿 매핑
- "응답 결과" 폴더: 생성된 스프레드시트 URL 찾기
- HTML 태그 제거
- diary 값으로 타입 결정 (0이 아니면 L, 0이면 X)

### 1.4 템플릿 시트 구조
템플릿 목록 스프레드시트는 템플릿 타입별로 3개 시트로 구성:

1. **일반 리커트 척도 시트**
   - 스크립트: `LegacyDataLoader.gs` → `loadLegacyTemplates()`
   - 내용: 179개 일반 템플릿 (xselect=1, xselect=2)
   - 컬럼: 템플릿ID, 템플릿명, 설명, 최소값, 최대값, 척도라벨(JSON), 질문리스트(JSON), 생성일, 상태

2. **자기 측면 검사 시트 (Template ID: -1)**
   - 스크립트: `LegacyDataLoader.gs` → `loadSpecialTemplate_MySide()`
   - 내용: 40개 단어 목록
   - 컬럼: 번호, 단어, 설명

3. **정서 자극 판단 과제 시트 (Template ID: -2)**
   - 스크립트: `LegacyDataLoader.gs` → `loadSpecialTemplate_Emotion()`
   - 내용: request_dic_list.csv에서 로드한 시나리오 데이터 (82개)
   - 컬럼: CSV 원본 컬럼 구조 유지

**참조 코드:**
- `getOrCreateTemplateSheet()`: "일반 리커트 척도" 시트 접근
- `getTemplateInfo()`: "일반 리커트 척도" 시트에서 템플릿 정보 조회
- `deleteExistingLegacyData()`: 시트별 상태 컬럼 인덱스 동적 결정

## 2. 문제 해결 이력

### 2.1 템플릿 정보 조회 실패
**원인:**
- 시트 이름 불일치
- 컬럼 인덱스 불일치

**해결:**
- 시트 이름 "템플릿 목록"으로 수정
- parseTemplateRow() 컬럼 인덱스 재매핑

### 2.2 파일명 파싱 실패
**원인:** 정규식 패턴 불일치 (`PKG(\d+)` vs `[PKG000010]`)

**해결:** `fileName.match(/\[PKG(\d+)\]/)` 변경

### 2.3 패키지 목록 시트 ID 오류
**원인:**
- CONFIG 객체 중복 선언
- 잘못된 스프레드시트 ID

**해결:**
- CSVtoSheetsConverter.gs에만 CONFIG 선언
- LegacyDataLoader.gs는 LEGACY_CONFIG 사용
- 패키지 목록 실제 ID로 변경

### 2.4 상태 컬럼 인덱스 불일치
**원인:**
- 패키지 시트 상태 컬럼이 인덱스 9인데 8에서 검색
- 이전 잘못된 구조(7개 컬럼)로 저장된 데이터

**해결:**
- 시트 이름별 상태 컬럼 인덱스 동적 결정
- 인덱스 6과 9 모두 체크

### 2.5 CSV 파싱 오류
**원인:** hello 컬럼의 여러 줄 텍스트를 `split('\n')`으로 단순 분리

**해결:** `Utilities.parseCsv()` 사용

### 2.6 HTML 태그 노출
**원인:** 패키지 설명에 HTML 태그 포함

**해결:** 정규식으로 HTML 태그 제거 및 공백 정리

## 4. 기술 노트

### 4.1 핵심 개념

**xselect 필드:**
- xselect=1: 공통 척도
- xselect=2: 개별 선택지

**연구 타입:**
- L (Longitude): 종단연구 - 응답 시점 여러 개
- X (Cross-sectional): 횡단연구 - 응답 시점 하나

**데이터 관계:**
```
패키지 → 템플릿 목록 → 안내 메시지 → 날짜 범위 → 스프레드시트 URL
템플릿 → 척도 정의 → 질문 목록
응답 데이터 → superstep (템플릿 인덱스) → step → sub_step → value
```

**상태 코드:**
- P (Protected): 레거시 데이터, 수정/삭제 불가
- A (Active): 활성 상태, 수정 가능
- D (Deleted): 삭제됨, 표시 안 함

### 4.2 인코딩
- 원본 SQL: EUC-KR
- 변환 CSV: UTF-8
- JSON: UTF-8
- Google Sheets: UTF-8

### 4.3 GAS 제약
- 실행 시간 제한: 6분
- 대응: 배치 처리 (3개씩)
- 진행 상황 저장: ScriptProperties
